| 元素类型 | Element Type  | 命名风格             | 示例                    |
| :------- | :------------ | :------------------- | :---------------------- |
| 类名     | Class         | PascalCase（大驼峰） | `MyClass`,`HttpRequest` |
| 模块     | Module        | 小写+下划线          | `my_module.py`          |
| 函数     | Function      | 小写+下划线          | `my_function()`         |
| 变量     | Variable      | 小写+下划线          | `my_variable`           |
| 常量     | Constant      | 全大写+下划线        | `MAX_SIZE`,`pi`         |
| 私有成员 | Private       | 单下划线前缀         | `_private_var`          |
| 强私有   | Name mangling | 双下划线前缀         | `__strong_private`      |

| 强私有/名称修饰/Name mangling：
| 在类的属性声明时，用双下划线前缀且末尾无下划线时，Python 解释器在类定义时会对该属性进行名称修饰，其真实属性名被改为`_<对应类的名称><原属性名>`，在外部访问或者访问其祖先类的该属性时要以`_<对应类的名称><原属性名>`的格式才能访问到真实的该属性。
| 子类未单独定义`<原属性名>`且在子类方法的内部直接访问`<原属性名>`时会出错，因为父类中访问的`<原属性名>`在外部看来实际名为`__<该父类的名字><原属性名>`。除了子类调用父类的方法中可以正常访问父类的`__<该父类的名字><原属性名>`。
| - `_<对应类的名称><原属性名>`格式例如：`_ClassName__prop_name`
